# Demo Script

Length: 3 to 5 minutes

## 1. Opening

"Our project is InsightSelf, a native Android self-exploration app for COMP7506C Smartphone Apps Development. The main idea is to use one unified profile to power several self-reflection modules: Bazi, Zodiac, questionnaires, and an integrated AI or template-based report."

## 2. Register

Open the app on the Android emulator.

"The app starts with username and password registration and login. The backend returns a bearer session token, so later profile, Bazi, Zodiac, questionnaire, report, and chat calls are tied to the authenticated user rather than trusting a raw user id from the client."

Register a new account.

## 3. Create Unified Profile

Show onboarding.

"After registration, the user creates one profile. This includes nickname, birth date, birth time, birth place, timezone, latitude and longitude, calendar type, cultural lens, and whether AI synthesis is enabled. The important design point is that this single profile drives every module in the app."

Submit the profile.

## 4. Home Dashboard

Show the Home tab.

"This is the main dashboard. It integrates profile overview, today's Zodiac board, Bazi highlight, assessment snapshot, integrated insight, the latest report summary, and a compact AI chat entry in the header. The dashboard can still show useful template text before a full provider-backed report is generated."

Point out the card layout and refresh button.

## 5. Generate Bazi Insight

Open the Bazi tab and tap Generate Bazi Insight.

"The Bazi module uses the 6tail lunar-java calendar library to calculate the four pillars. It then produces five element scores from visible and hidden stems. The radar chart visualizes Wood, Fire, Earth, Metal, and Water. We also show conclusion, evidence, and suggestions in separate sections."

Mention the disclaimer:

"This is for self-reflection and course demonstration. The calendar calculation is library-backed, while the interpretation remains soft guidance rather than professional destiny analysis."

## 6. Zodiac Daily Board

Open the Zodiac tab.

"The Zodiac module uses Swiss Ephemeris to calculate the user's natal chart and today's transit positions. The daily board scores emotion, communication, and action from selected personal-planet aspects, so the same user gets the same result on the same date."

Open Relationship Matching if time allows.

"Relationship matching can use the target's birthday only, or a full target birth time, timezone, latitude, and longitude. It compares astrology aspects and optionally uses the user's latest MBTI-style result."

## 7. Complete One Assessment

Open Assessments and choose one assessment, such as Big Five or MBTI-style Inventory.

"The assessment hub keeps the API-compatible BFI10, MBTI, and ATTACHMENT type codes, but the user interface presents BFI10 as Big Five. The backend seeds a 20-item IPIP-style Big Five form, a 32-item MBTI-style preference inventory, and a 9-item ECR-RS-style attachment form. Each question uses a 1 to 5 response scale, and the backend stores scores plus instrument details."

Submit the assessment and show the result label and score details.

## 8. Generate Integrated Report

Return to Home or open the AI Report screen. Tap Generate / Refresh.

"The report module can call Tongyi Qianwen through an OpenAI-compatible endpoint when `QWEN_API_KEY` is configured. If there is no API key, the backend uses a stable template fallback. If the key is configured but the provider fails, the backend reports that failure instead of silently pretending a template report was generated by AI."

Show the report.

If time allows, open the AI chat entry from Home and send two related questions.

"The chat endpoint uses the same compact InsightSelf context as the report, plus recent conversation history. It returns a session id so the Android client can continue a multi-turn conversation without sending raw dashboard snapshots or prior report text to the model."

## 9. Closing

"InsightSelf demonstrates a complete Android and Spring Boot MVP: native Android UI, Retrofit networking, database-backed profile and results, library-backed Bazi and astrology calculations, structured questionnaire scoring, and documented APIs. The core design is Single Profile plus structured interpretation, so each module contributes to one coherent dashboard."

## Backup Notes

If the AI API is not configured, say:

"This report is generated by the template fallback, which is intentional for demo stability."

If the AI API is configured, say:

"This report source is QWEN. The Android app receives the report text and source metadata, while the compact generation snapshot stays on the backend."

If the backend is not reachable, show:

"The app displays network error states rather than crashing. For emulator testing, the backend URL is `http://10.0.2.2:8080/`."
